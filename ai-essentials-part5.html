Part 5

Regularization Techniques

Beginner-friendly · Neural Networks · Fighting Overfitting

In Parts 1 and 2 we learned that overfitting is the enemy of good models. In Part 4 we saw how networks learn via backpropagation. Now the question is — how do we stop a network from overfitting while it learns? The answer is regularization.

🛡️ What is regularization?

Regularization is a set of techniques that constrain a model during training to prevent it from memorizing the training data. Instead of fitting every detail, the model is forced to learn only the most important patterns — patterns that will generalize to new data.

Think of it like studying for an exam. Memorizing every practice question is overfitting. Understanding the underlying concepts is regularization — it prepares you for questions you've never seen before.

🔧 The three main regularization techniques

DO

Dropout

Most popular in deep learning
During each training step, randomly "drop" (set to zero) a percentage of neurons. This forces the remaining neurons to learn redundant representations — the network can't rely on any single neuron and becomes more robust.

L2

L2 Regularization

Also called Weight Decay or Ridge
Adds a penalty to the loss function proportional to the square of the weights. This discourages large weights — keeping the model simple and preventing extreme over-reliance on any one feature.
Loss = Original Loss + λ × Σw²
λ (lambda) controls the strength of regularization. Larger λ = stronger penalty = simpler model.
L1

L1 Regularization

Also called Lasso
Similar to L2 but uses the absolute value of weights instead of squares. L1 pushes some weights all the way to zero — effectively removing those features from the model. This produces a sparse model.
Loss = Original Loss + λ × Σ|w|
L1 is useful for feature selection — it automatically discards unimportant inputs by zeroing their weights.

⚖️ L1 vs L2 — key differences

L1 (Lasso)

Produces sparse models. Some weights become exactly zero. Good for feature selection. Works well when only a few features matter.

L2 (Ridge)

Keeps all weights small but non-zero. Good for preventing overfitting when all features contribute. More stable than L1.

📊 When to use which technique

TechniqueBest forTypical rate
DropoutLarge neural networks, image/speech models20–50% of neurons
L2 (Ridge)Most neural networks as default regularizerλ = 0.0001–0.01
L1 (Lasso)When you want automatic feature selectionλ = 0.0001–0.01

✅ Regularization checklist — before training

Add Dropout after large dense layers (rate 0.2–0.5)
Use L2 weight decay in your optimizer settings
Monitor validation loss — if it rises while train loss falls, you need more regularization
Use Early Stopping to halt training when validation loss stops improving
Collect more data — regularization helps but more data is always better

💡 Final takeaway

Overfitting is not bad luck — it is predictable and preventable. Dropout forces your network to be robust. L2 keeps weights small. L1 removes noise features. Use them together and your model will generalise far better to data it has never seen. These techniques are what separate a toy model from a production-ready one.
