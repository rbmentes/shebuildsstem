How Backpropagation Works

Beginner-friendly · Neural Networks · The Math Behind Learning

In Part 3 we learned that activation functions give neural networks their power. But how does a network actually learn? How does it know which weights to adjust? The answer is backpropagation — the algorithm that makes deep learning possible.

🧠 The big picture

Backpropagation is how a neural network learns from its mistakes. It measures the error in the output, then works backwards through the network — layer by layer — adjusting weights to reduce that error. Repeat this thousands of times and the network gradually improves.

🔄 The four steps of training

1

Forward pass

Input data flows through the network layer by layer. Each layer applies weights, biases, and activation functions until an output prediction is produced.

2

Compute the loss

The prediction is compared to the true label using a loss function (e.g. Mean Squared Error or Cross-Entropy). This gives a single number measuring how wrong the network was.

3

Backward pass — compute gradients

Using the chain rule from calculus, the gradient of the loss is computed with respect to every weight in the network — starting from the output and working backwards.

4

Update weights

Each weight is nudged in the direction that reduces the loss. The size of the nudge is controlled by the learning rate.

🔗 The chain rule — the math behind it

Backpropagation relies on the chain rule from calculus. To find how a weight deep in the network affects the final loss, we multiply the gradients along each step of the path:

∂Loss/∂w = ∂Loss/∂output × ∂output/∂hidden × ∂hidden/∂w

Think of it like a chain of responsibility. The last layer is most directly responsible for the error. Earlier layers are responsible through their effect on later layers. The chain rule links them all together.

📉 Gradient descent — how weights are updated

Once we have the gradients, we update each weight by subtracting a small fraction of the gradient:

w = w - η × ∂Loss/∂w

Where η (eta) is the learning rate — a small number like 0.001 that controls how big each update step is.

⚠️ The vanishing gradient problem

In very deep networks, gradients can become extremely small as they travel backwards through many layers. By the time they reach the early layers, the updates are nearly zero — those layers barely learn. This is called the vanishing gradient problem and it's one reason ReLU became popular over Sigmoid.

💡 Final takeaway

Backpropagation is the engine of deep learning. Forward pass makes a prediction, loss measures the error, backward pass figures out who is responsible, and gradient descent fixes it. Repeat millions of times — and the network learns. Understanding this loop is the foundation for understanding everything else in deep learning.
