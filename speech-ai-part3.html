Understanding Spectrograms & Convolution

Beginner-friendly · Deep Learning · Speech Processing

In Part 2, we explored how CNNs process speech by treating audio as an image. But what exactly is that "image"? That's what this part is all about.

📊 What is a spectrogram?

A spectrogram is a visual representation of how the frequency content of a signal changes over time. It is generated by applying Short-Time Fourier Transform (STFT) to small overlapping frames of audio — turning a 1D waveform into a 2D time-frequency map.

🧠 Why spectrograms are powerful

Preserve both time and frequency information simultaneously
Make local frequency transitions visible as 2D patterns
Allow CNNs to scan speech exactly like scanning an image
Different languages and phonemes produce visually distinct patterns

🔍 What is convolution?

Convolution is a mathematical operation where a small filter (kernel) slides across an input matrix computing weighted sums at each position. This is how CNNs detect patterns — edges in images, frequency transitions in speech.

⚙️ Key CNN parameters

📐
Kernel size
Size of the sliding filter (e.g. 3×3 or 5×5). Larger = broader pattern detection.
👣
Stride
How many steps the kernel moves each time. Larger stride = smaller output feature map.
🔲
Padding
Adds border values so edge information is not lost during convolution.

🚀 Why this matters

Understanding spectrograms and convolution helps us design better speech models. When we know what the model is actually "seeing" and how it scans for patterns, we can make smarter choices about architecture and preprocessing.
