How Do Models Actually Learn?

Spring 2026 · Beginner-friendly · Loss Functions & Gradient Descent

In Week 1 we learned what machine learning is — models that find patterns in data. But there's a question I had immediately after: how exactly does a model get better? How does it know when it's wrong, and how does it fix itself? This week answers that.

🎯 Step 1 — measuring how wrong the model is

A loss function (also called a cost function) measures the difference between what the model predicted and what the correct answer actually was. It gives a single number — the lower the loss, the better the model is performing.

Example — predicting house price (in $1000s)
Model predicted
$720k
720
Actual price
$850k
850
Error (loss)
$130k
130

The model was off by $130k. The loss function turns this mistake into a number the model can learn from. The goal of training is to make this number as small as possible.

📊 Common loss functions

📏

Mean Squared Error (MSE)

Used for: Regression problems

Takes the average of squared differences between predictions and actual values. Squaring means large errors are punished more heavily.

MSE = (1/n) × Σ(y - ŷ)²
🎯

Cross-Entropy Loss

Used for: Classification problems

Measures how different the predicted probability distribution is from the true labels. Higher confidence in the wrong answer = higher loss.

Loss = -Σ y × log(ŷ)

⛰️ Step 2 — gradient descent

Now we know how wrong the model is. But how does it get better? The answer is gradient descent — the algorithm that adjusts the model's weights to reduce the loss.

Imagine you're standing on a hilly landscape in the dark and you want to reach the lowest point. You can't see far — but you can feel the slope under your feet. So you take a small step downhill. Then another. Then another. Eventually you reach the bottom. That's gradient descent.

Loss landscape — gradient descent finds the minimum

🎚️ The learning rate — how big are the steps?

The learning rate controls how large each step down the hill is. It's one of the most important settings in ML training — too small and training is painfully slow, too large and you overshoot the minimum entirely.

🐌

Too small (e.g. 0.00001)

Takes tiny steps. Training is very slow and may never converge in reasonable time.

✓

Just right (e.g. 0.001)

Takes steady steps. Converges smoothly to a good minimum. This is typical for most models.

💥

Too large (e.g. 0.1)

Overshoots the minimum. Loss bounces up and down and never settles. Model fails to learn.

📅 What is an epoch?

One epoch = the model has seen every training sample once. We usually train for many epochs — each pass through the data the model improves a little:

Epoch 1
22%
Epoch 5
55%
Epoch 10
78%
Epoch 20
91%
Epoch 40
97%

💡 What I learned this week

Learning in ML is really just one loop: predict → measure the mistake → take a small step to fix it → repeat. The loss function is the report card, gradient descent is the study session, and the learning rate controls how fast you study. Understanding this loop is the foundation for understanding every ML model that exists.
