Building a CNN for Speech Classification

Hands-on · Python · PyTorch · Speech Processing

In Parts 1–3 we covered audio features, MFCCs, CNNs, spectrograms, and convolution mathematically. Now we actually build one. This part is hands-on — we write a complete CNN in PyTorch that takes audio, extracts log-Mel features, and classifies speech into categories.

🗺️ The full pipeline

Before writing any code, here is the complete flow from raw audio file to classification output:

→

→
📊Log-Mel spectrogram

📊 Step 2 — feature extraction

We convert each audio file into a log-Mel spectrogram using torchaudio. This is the "image" the CNN will learn from.

Python — log-Mel feature extractor def extract_log_mel(waveform, sample_rate, n_mels=64, n_fft=1024, hop_length=512): # Step 1: compute Mel spectrogram mel_transform = T.MelSpectrogram( sample_rate=sample_rate, n_fft=n_fft, hop_length=hop_length, n_mels=n_mels ) mel_spec = mel_transform(waveform) # Step 2: convert to log scale (adds small value to avoid log(0)) log_mel = torch.log(mel_spec + 1e-9) # Step 3: fix length — pad or trim to 128 frames target_length = 128 if log_mel.shape[-1] < target_length: log_mel = torch.nn.functional.pad(log_mel, (0, target_length - log_mel.shape[-1])) else: log_mel = log_mel[..., :target_length] return log_mel # shape: (1, 64, 128) # Test it with a generated waveform dummy_wave = torch.randn(1, 16000) # 1 second at 16kHz features = extract_log_mel(dummy_wave, sample_rate=16000) print("Feature shape:", features.shape) # (1, 64, 128)

🗂️ Step 3 — build the dataset class

Python — custom dataset class SpeechDataset(Dataset): def __init__(self, file_paths, labels, sample_rate=16000): self.file_paths = file_paths self.labels = labels self.sample_rate = sample_rate def __len__(self): return len(self.file_paths) def __getitem__(self, idx): waveform, sr = torchaudio.load(self.file_paths[idx]) # Resample if needed if sr != self.sample_rate: resampler = T.Resample(sr, self.sample_rate) waveform = resampler(waveform) # Use only first channel (mono) waveform = waveform[0].unsqueeze(0) features = extract_log_mel(waveform, self.sample_rate) label = torch.tensor(self.labels[idx], dtype=torch.long) return features, label

🧠 Step 4 — the CNN architecture

Our CNN has 3 convolutional blocks followed by fully connected layers. Each block learns progressively more abstract frequency-time patterns.

Conv2D

Convolutional layer × 3 blocks

Detects local patterns — frequency edges, temporal bursts, phoneme shapes

BatchNorm

Batch normalization

Stabilizes training by normalizing layer inputs — speeds up convergence

MaxPool

Max pooling

Reduces spatial dimensions — keeps strongest activations, reduces computation

Dropout

Dropout (0.3)

Randomly disables neurons during training — prevents overfitting

Linear

Fully connected layers

Combines all learned features into a final classification decision

Python — CNN model class SpeechCNN(nn.Module): def __init__(self, num_classes=2): super().__init__() # Block 1: (1, 64, 128) → (32, 32, 64) self.block1 = nn.Sequential( nn.Conv2d(1, 32, kernel_size=3, padding=1), nn.BatchNorm2d(32), nn.ReLU(), nn.MaxPool2d(2), nn.Dropout(0.2) ) # Block 2: (32, 32, 64) → (64, 16, 32) self.block2 = nn.Sequential( nn.Conv2d(32, 64, kernel_size=3, padding=1), nn.BatchNorm2d(64), nn.ReLU(), nn.MaxPool2d(2), nn.Dropout(0.2) ) # Block 3: (64, 16, 32) → (128, 8, 16) self.block3 = nn.Sequential( nn.Conv2d(64, 128, kernel_size=3, padding=1), nn.BatchNorm2d(128), nn.ReLU(), nn.MaxPool2d(2), nn.Dropout(0.3) ) # Classifier self.classifier = nn.Sequential( nn.Flatten(), nn.Linear(128 * 8 * 16, 256), nn.ReLU(), nn.Dropout(0.4), nn.Linear(256, num_classes) ) def forward(self, x): x = self.block1(x) x = self.block2(x) x = self.block3(x) return self.classifier(x) # Test forward pass model = SpeechCNN(num_classes=2) dummy = torch.randn(4, 1, 64, 128) # batch of 4 output = model(dummy) print("Output shape:", output.shape) # (4, 2) — 2 classes

🏋️ Step 5 — training loop

Python — training def train(model, loader, optimizer, criterion, device): model.train() total_loss, correct, total = 0, 0, 0 for features, labels in loader: features, labels = features.to(device), labels.to(device) optimizer.zero_grad() outputs = model(features) loss = criterion(outputs, labels) loss.backward() optimizer.step() total_loss += loss.item() correct += (outputs.argmax(1) == labels).sum().item() total += len(labels) return total_loss / len(loader), correct / total * 100 device = torch.device('cuda' if torch.cuda.is_available() else 'cpu') model = SpeechCNN(num_classes=2).to(device) optimizer = optim.Adam(model.parameters(), lr=0.001) criterion = nn.CrossEntropyLoss() for epoch in range(20): loss, acc = train(model, train_loader, optimizer, criterion, device) if (epoch + 1) % 5 == 0: print(f"Epoch {epoch+1}: Loss={loss:.4f} Acc={acc:.1f}%")

✅ Tips for better results

Always resample all audio to the same sample rate (16kHz is standard)
Normalize waveform amplitude before feature extraction
Use data augmentation — add noise, time shift, pitch shift — especially with small datasets
Monitor validation loss not just training loss — if it rises while train loss falls you're overfitting
Cache precomputed spectrograms to speed up training significantly
Start with a small model and add complexity only if performance plateaus
