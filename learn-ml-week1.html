What Is Machine Learning?

Spring 2026 · Beginner-friendly · From an engineering student

📌 Why I'm starting this series

This semester, I'm taking Machine Learning for the first time. Instead of just studying for exams, I decided to document what I learn — step by step.

This series is not written as an expert. It's written as an engineering student building understanding from the ground up.

🧠 What is machine learning?

Machine Learning is a field of study that gives computers the ability to learn patterns from data — without being explicitly programmed for every rule.

Instead of writing rules manually, we provide data, examples, and sometimes expected outputs. The model learns the relationship on its own.

🔍 Types of machine learning

🏷️

Supervised Learning

Learns from labeled data — every input has a known correct output.

e.g. Deepfake detection
🔎

Unsupervised Learning

Finds structure in unlabeled data — no correct answers given.

e.g. Audio clustering
🎮

Reinforcement Learning

Learns by interacting with an environment and receiving rewards.

e.g. Game-playing AI

📊 Regression vs classification

In supervised learning, two main task types appear:

Regression

Output is a continuous number — e.g. house price, temperature.

Classification

Output is a category — e.g. real / fake, spam / not spam.

In my deepfake detection project, I worked on classification — predicting whether a voice sample is real or AI-generated.

📂 Dataset, features and labels

Every ML dataset is built from two things:

X

Features — the input variables the model learns from. In speech: MFCCs, spectrogram values, pitch.

y

Labels — the target outputs we want to predict. In speech: Real or Fake.

🎯 Training vs test set

We always split our data before training:

Typical 80/20 data split
Training set — 80%
Test — 20%
Model learns patterns here Model is evaluated here

This split helps us measure generalization — the model's ability to perform well on data it has never seen before.

⚡ Why generalization matters

A model that memorizes training data but fails on new data is not useful. This connects directly to overfitting — a concept I explained in my AI Essentials series.

💡 What I realized this week

Machine Learning is not magic. It is structured pattern recognition powered by mathematics. Understanding the vocabulary is the first step — and now you have it.
