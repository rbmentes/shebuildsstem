Part 3

Activation Functions

Beginner-friendly · Neural Networks · Deep Learning

In Parts 1 and 2 we covered overfitting and the bias-variance tradeoff. Now we go deeper into how neural networks actually work — starting with one of the most important building blocks: activation functions.

🧠 Why do we need activation functions?

Without activation functions, a neural network — no matter how many layers it has — is just doing linear math. It can only draw straight lines through data. Activation functions introduce non-linearity, allowing networks to learn complex patterns.

Without activation

Every layer is just a linear transformation. Stack 100 layers and it still behaves like a single layer — useless for complex problems.

With activation

Each layer can learn non-linear patterns. The network can model curves, boundaries, and complex structures in data.

⚡ The three most important activation functions

R

ReLU — Rectified Linear Unit

f(x) = max(0, x)
The most popular activation function today. Outputs the input directly if positive, otherwise outputs zero. Simple, fast, and works surprisingly well.
Output range: 0 to +∞
Fast to compute
Can cause "dead neurons"
σ

Sigmoid

f(x) = 1 / (1 + e⁻ˣ)
Squashes any input into a value between 0 and 1 — perfect for binary classification outputs (probability). Historically popular but now mostly used only in output layers.
Output range: 0 to 1
Good for probabilities
Vanishing gradient problem
T

Tanh — Hyperbolic Tangent

f(x) = (eˣ - e⁻ˣ) / (eˣ + e⁻ˣ)
Similar to Sigmoid but outputs values between -1 and 1. Because it is centred at zero, it often trains better than Sigmoid in hidden layers. Still suffers from vanishing gradients at extremes.
Output range: -1 to +1
Zero-centred output
Slow at extreme values

💀 The dying ReLU problem

If a neuron's input is always negative, ReLU always outputs zero — and the neuron never updates during training. It becomes "dead." This is solved by Leaky ReLU which allows a small negative output instead of zero: f(x) = max(0.01x, x).

📊 When to use which activation

FunctionUse it whenAvoid when
ReLUHidden layers — default choice for most networksYou have many dead neurons
SigmoidBinary classification output layerHidden layers — causes vanishing gradients
TanhHidden layers when zero-centred output mattersVery deep networks — still has vanishing gradient

💡 Final takeaway

Activation functions are what give neural networks their power. Without them, deep networks would be mathematically equivalent to a single layer. ReLU is your default — fast, simple, effective. Sigmoid lives at the output for binary problems. Tanh is the zero-centred alternative for hidden layers.
