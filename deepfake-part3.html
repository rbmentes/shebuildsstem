📊 Feature extraction — log-Mel spectrogram

Instead of feeding raw audio waveforms to the model, I first convert audio to a log-Mel spectrogram — a 2D time-frequency representation that captures how energy is distributed across frequency bands over time. This is the same feature representation we covered in the Speech AI series.

The conversion pipeline:

Step 1 — Short-Time Fourier Transform X(m,k) = Σ x[n] · w[n−mH] · e^(−j2πkn/N)
Step 2 — Mel scale mapping Mel(f) = 2595 · log₁₀(1 + f / 700)
Step 3 — Log compression S_log-mel = log(S_mel + ε) ε prevents log(0)

🧠 Why LCNN?

LCNN stands for Light Convolutional Neural Network. It was originally proposed for face recognition but proved highly effective for audio spoof detection due to its efficient architecture and a special activation function called Max-Feature-Map (MFM).

Standard ReLU

Outputs max(0, x) — simply removes negative values. Treats all features equally regardless of their discriminative value.

MFM activation (LCNN)

Takes two feature maps and outputs the element-wise maximum. This selects the most informative features and suppresses less useful ones — better for detecting subtle artifacts.

MFM activation function MFM(a, b) = max(a, b)

The key insight is that deepfake artifacts are subtle — they appear in specific frequency regions and temporal patterns. MFM helps the network focus on these discriminative patterns by competitively selecting between feature map pairs, effectively suppressing noise and irrelevant responses.

🏗️ LCNN architecture layers

Conv

Convolutional layers

Detect local time-frequency patterns in the spectrogram — frequency transitions, energy bursts, artifact signatures

MFM

Max-Feature-Map activation

Competitive selection between feature map pairs — suppresses uninformative features, enhances discriminative ones

MaxPool

Max pooling

Reduces spatial dimensions progressively — retains strongest activations, improves computational efficiency

FC

Fully connected layers

Combines all extracted features into a final binary classification: genuine (0) or fake (1)

📏 The evaluation metric — EER

I evaluated the model using the Equal Error Rate (EER) — the point where False Acceptance Rate (FAR) equals False Rejection Rate (FRR).

Equal Error Rate definition EER = FAR(τ*) = FRR(τ*) where τ* is the optimal decision threshold

A lower EER is better — it means the system makes fewer errors of both types. An EER of 0% would be a perfect system.

💡 Why LCNN works for deepfake detection

Deepfake artifacts in audio are like tiny imperfections in a painting — subtle but present. LCNN's MFM activation is specifically designed to find and amplify these differences while ignoring natural speech variation. Combined with log-Mel spectrograms that capture the same frequency information humans use to perceive speech, the system can learn what "fake" looks like in a way that standard CNNs often miss.
